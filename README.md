## 1. 시나리오 제목
<br>
대학생 맞춤형 '갓생 살기' 투두 메이트(Todo-mate) 시스템



## 2. 시나리오 (5~10줄)
<br>
이 프로그램은 대학생들이 하루의 목표를 설정하고 성취도를 시각적으로 확인할 수 있는 **'자기관리 루틴 시스템'**입니다.<br>
사용자는 자신의 이름, 오늘 날짜, 그리고 계획한 총 할 일의 개수와 실제 완료한 개수를 입력합니다. 시스템은 이를 바탕으로 오늘의 최종 달성률(%)을 계산하여 출력합니다.
단순히 수치만 보여주는 것이 아니라, 달성률 구간에 따라 맞춤형 응원 메시지를 제공하는 기능을 포함합니다. 향후에는 할 일 목록을 직접 배열(List)로 저장하고, 주간 평균 달성률을 산출하거나 카테고리별(공부, 운동 등)로 관리하는 기능을 추가하여 완성도 높은 개인 비서 프로그램을 만들 계획입니다.


## 3. 예상 기능 및 메뉴 (최소 5개)
<br>
1. 사용자 프로필 및 오늘 날짜 설정<br> 
2. 할 일 목록 추가 및 개수 관리<br> 
3. 실시간 달성률(%) 계산 및 출력<br>  
4. 달성 점수에 따른 동기부여 등급 판별<br>  
5. 주간/월간 누적 데이터 조회 및 분석<br>  
6. 프로그램 종료

## 🟨 [3차 과제: V3.0] 무한 루프와 메뉴 시스템 (반복문) - 향후 작성 예정

✨ **3차 과제 업데이트 내용:**
* **`while True` 기반 무한 루프 메뉴 도입:** 프로그램을 종료(`4번`)하기 전까지 무한히 반복 실행되며 사용자 입력을 대기하는 메인 메뉴 시스템을 구축했습니다.
* **기능별 독창적 함수 분리 (`def` 활용):** 기존 2차 과제 기능을 `input_todo()`, `show_todos()`, `complete_todo()`의 최소 3개 이상의 개별 함수로 쪼개어 가독성을 높였습니다.
* **매개변수 및 반환값(return) 패턴 적용:** 조회 함수에는 리스트를 인자로 전달하고, 완료 처리 함수에서는 처리 결과를 성공/실패 여부에 따라 `1` 또는 `0`으로 반환받도록 설계했습니다.
* **`global` 변수를 통한 상태 관리:** 프로그램 전체에서 유지되어야 하는 누적 완료 개수(`completed_count`)를 `global` 키워드를 활용해 함수 내부에서 제어 및 업데이트했습니다.

🤖 **AI 파트너십 과정**
1. **투두메이트 아키텍처 및 함수 분리 설계**
* **프롬프트 요약:** "기존에 작성하던 투두메이트 프로그램에 무한 루프 메뉴를 적용하고, 기능을 최소 3개의 독립된 함수로 분리해줘. 이때 매개변수, return문, global 키워드를 각각 1회 이상 무조건 포함해야 하고 메인 코드는 간결해야 해."
* **적용 내용:** AI의 추천을 받아 전역 변수로 투두 리스트와 완료 카운트를 선언했습니다. 입력, 조회, 완료 처리를 완전한 독립 함수로 분리하여 메인 로직(`while`문 내부)에는 오직 깔끔한 조건문과 함수 호출만 남겨 '목차형 구조'를 완성했습니다.
2. **리턴값 기반의 예외 및 상태 제어 최적화**
* **프롬프트 요약:** "complete_todo 함수에서 유저가 번호를 잘못 입력하거나 리스트가 빌 때의 예외 처리를 return문 조건에 맞춰서 보완해줘."
* **적용 내용:** 입력값이 범위를 벗어나거나 빈 값일 때 `return 0`을 반환하고, 정상적으로 할 일이 완료되었을 때만 완료 카운트를 올리며 `return 1`을 반환하도록 분기 처리를 구조화했습니다.

🛠️ **Troubleshooting & 기술 회고:**
1. **문제 1: 함수 내부에서 전역 변수(Count) 수정 시 발생하는 UnboundLocalError**
   * **원인:** `complete_todo()` 함수 내에서 전역 변수인 `completed_count` 값을 직접 수정(`+= 1`)하려 시도했으나, 파이썬이 이를 지역 변수로 오인하여 할당 전에 참조되었다는 에러가 발생했습니다.
   * **해결:** 함수 도입부에 `global completed_count` 키워드를 명시적으로 선언함으로써, 해당 변수가 지역 변수가 아닌 프로그램 최상단의 전역 변수임을 컴퓨터에게 명확히 알려주어 문제를 해결했습니다.

2. **문제 2: 메인 로직 가독성 저하 및 감점 요인 제거**
   * **원인:** 처음 구조를 잡을 때 `while`문 내부에 메뉴 출력문과 입력값 조건 처리가 길게 나열되어, '메인 로직은 함수 호출 위주로 간결하게 구성할 것'이라는 과제 제약 사항에 위배될 위험이 있었습니다.
   * **해결:** 사용자 메뉴 출력과 호출 부분을 제외한 모든 세부 연산 및 출력 가이드라인을 각각의 함수(`input_todo`, `show_todos`, `complete_todo`) 내부로 전부 밀어 넣어 메인 코드를 목차처럼 직관적으로 리팩토링했습니다.

3. **문제 4: 코드 가독성 과정에서 발생한 'statua' 오타로 인한 런타임 에러**
   * **원인:** 할 일 출력창의 완료 상태를 조율하는 과정에서, 변수명을 `status`가 아닌 **`statua`**로 잘못 입력하는 단순 오타가 발생했습니다. 이로 인해 파이썬이 존재하지 않는 지역 변수를 호출하려다 프로그램이 비정상 종료되었습니다.
   * **해결:** 에러 로그를 기반으로 `else:` 블록 내의 철자 오류 위치를 정확히 찾아내어 `status`로 수정했고, 실행 흐름에 관계없이 변수가 안정적으로 선언 및 바인딩되도록 조치했습니다

  * **프롬프트 요약:** "while True 무한 루프를 돌려서 프로그램을 만드는데, 사용자가 4번을 눌러서 종료하기 전까지 데이터가 안전하게 유지되다가 종료 시점에 메모리가 깔끔하게 해제되는 구조가 맞는지 로직
    검증해 줘."
  * **적용 내용:** 무한 루프 제어 흐름과 변수의 스코프(범위)에 대해 AI와 크로스 체크를 진행함. 종료 메뉴(4번) 선택 시 하위 함수들이 안전하게 리턴된 후 break가 호출되어 루프를 탈출하는 전체 실행 흐름의
    안정성을 검증받아 반영함.

### **🛠️ Troubleshooting & 기술 회고:**
1. **문제 1: 파이썬 함수 내부의 들여쓰기(IndentationError) 및 정의되지 않은 변수 오류**
   * **원인:** 코드를 수정하고 직접 타이핑하는 과정에서 `show_todos` 함수 내의 중첩 `for item in todos:` 블록이 함수의 내부 들여쓰기 라인 밖으로 밀려나 맨 앞으로 배치됨. 이로 인해 파이썬 인터프리터가 해당 구문을 함수 외부에 있는 것으로 오인하여 매개변수인 `todos`를 인식하지 못하고 `Pylance(reportUndefinedVariable)` 에러를 발생시킴.
   * **해결:** 에러가 발생한 `for`문부터 하위 로직 전체를 마우스로 블록 지정한 뒤, `Tab` 키를 활용해 함수 내부 스페이스 4칸 라인에 맞춰 수직 정렬함. 들여쓰기 구조를 복구하자마자 변수 미정의 에러가 마법처럼 동시에 해결됨.

2. **문제 2: 전역 변수명 오타로 인한 로직 마비**
   * **원인:** 할 일 등록 기능 수행 중 할 일의 고유 ID를 축적·관리하는 전역 변수를 불러오는 과정에서 `global current_id`를 `global currend_id`로 끝 글자를 잘못 입력하는 사소한 오타가 발생함. 선언된 변수와 명칭이 달라 프로그램이 정상 작동하지 않음.
   * **해결:** 코드 상단의 초기 선언부와 함수 내부의 전역 변수 명칭을 하나씩 대조하여 `currend_id`를 `current_id`로 정확하게 철자를 수정함.

3. **문제 3: 표 출력 시 텍스트 및 별(★) 기호 정렬 어긋남 문제**
   * **원인:** 목록을 조회했을 때 '미완료' 글자 바로 옆에 우선순위 별 모양(`★`)이 바짝 붙어서 출력되거나 줄 격자가 삐뚤어지는 현상 발생. 각 데이터가 차지하는 칸의 너비(포맷팅 서식)를 완벽하게 통일해 주지 않아 발생한 UI 레이아웃 꼬임 문제였음.
   * **해결:** 데이터의 각 인덱스를 순회할 때 `idx == 2`인 상태 칸에는 `{val:<10}`, `idx == 3`인 우선순위 칸에는 `{'★' * val:<8}`과 같이 일정한 서식 너비를 지정해 줌으로써 미완료 글자와 별 사이에 예쁜 세로 격자 정렬이 유지되도록 수정함.
